우리의 ux-design-lead 서브에이전트에

  Please go aghead and implement this design system

  - install fonts and dependencies
  - create resuable components
  - update the css, ts(design token)
  - any other tasks related to the design system

  "Stitch MCP 도구를 사용해서 각 페이지의 HTML을 다운로드해주세요. 그러면 스타일링, 레이아웃, 애니메이션 등에 대한 중요한 컨텍스트를 얻을 수 있습니다."

라는 프롬프트를 실행하면 실제로 stitch mcp는 만들어진 페이지를 html 파일로 내려받게되어있어. 

이 데이터를 기반으로 작업하는 디렉토리의 디자인 시스템관련 파일들(css, ts, etc)에 적용하고 
ui 작업시 이 html의 페이지와 1:1 매핑하여 올바른 디자인을 구성할 수 있도록 해야함.

물론 이 작업은 MCP가 활성화 되었는지 검증 후에 작업해야함.

  그리고 디자인 작업을 수행할때 docs/design-system 디렉토리의 token.josn 과 문서를 읽어 참조하는데, 여기에 이미지
  파일도 존재하면 분석하라고 지침을 넣어야함.

  작업하고자하는 프로젝트 디렉토리의 package.json 파일을 읽고 추가 라이브러리의 패턴을 사용해야함, 추가할
  라이브러리가 있다면 추가하면됨.
  이 지시가 겹쳐서 토큰을 낭비되는 부분이 있거나 현재 설계에서 충돌을 일으킬 부분을 찾아보셈.

  이 서브에이전트의 재설계가 필요함. 
  harness pipeline 스킬 워크플로우에 자동 로딩되어 실행되는것 외에 사용자가 이 서브에이전트를 호출 시 
  작동도 가능해야함.

예를 들어 docs/design-system 디렉토리를 직접 생성하고 문서와 token.json을 만들고 싶어서 프롬프트를 직접 입력할수도 있어.
그리고 사용자가 특정 페이지느 컴포넌트들에대해서 디자인 리뷰 요청을할 수도 있어. 그리고 사용자가 특정 디자인 요청이나 수정을 요청할 수도 있어.

위 내용들은 필요한 부분들을 정리해놓은건데 너무 복잡해서 정리가 잘안돼. 순서를 잘 정해야할 것 같아. 
모호하거나 구체적인 정의가 필요하면 나와 논의하자.
